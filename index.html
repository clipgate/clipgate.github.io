The Problem

Your clipboard is a black hole.
You copy dozens of things per session. Only the last one survives.

Your clipboard holds one thing. You copy dozens per session — errors get overwritten by paths, commands vanish, that stack trace from 3 minutes ago is gone forever. Clipboard history tools exist, but they show a flat list sorted by time. You don't need history. You need clipboard intelligence — knowing what each item is and retrieving by type. And when you're working with AI assistants, instead of copy-pasting errors one by one, cg pack bundles them with metadata and pipes directly to Claude, Codex, or Cursor.

Philosophy

Design principles

Local First

Everything runs on your machine. SQLite storage. No cloud. No telemetry. No accounts.

Invisible Until Needed

Minimal output by default. One-line confirmation on stderr. Add --quiet for true silence.

Secrets Are Sacred

API keys auto-detected. Memory-only vault. 5-minute TTL. Zero bytes on disk, ever.

AI-Native

Built for AI workflows from day one. Context packing, MCP server, structured output formats.

Roadmap

Where Clip Gate is headed — and how far it's come.

Phase 1 — Core CLI Complete

Copy, paste, list, search. SQLite storage with WAL mode. ULID-based item IDs. Priority-based content classification and confidence scores.

Phase 2 — Secrets & MCP Complete

Secret detection with 15 patterns + Shannon entropy. In-memory TTL vault — secrets never hit disk. MCP server with 6 tools over stdio JSON-RPC 2.0.

Phase 3 — AI Handoff & Shell UX Complete

Context packer with text, JSON, and markdown formats. Shell integration with 7 aliases and helper functions. Interactive fzf picker. Doctor health checks. 220+ tests passing.

Phase 4 — Platform & Packaging Complete

Linux clipboard adapters (Wayland + X11). Homebrew formula + .deb/.rpm packaging. cg push / cg pop named stacks. 6 new content types. Golden packer tests. VHS terminal recordings. 220+ tests.

Phase 5 — Intelligence Up Next

Semantic search with local embeddings (no cloud). Auto-tagging with git repo and branch context. Session-aware grouping. Frequency analysis for repeat pastes. cg suggest — proactive clipboard recommendations.

Phase 6 — Ecosystem Planned

Plugin system for custom classifiers. Team sharing via encrypted sync. VS Code and JetBrains extensions. Web dashboard. cg export to Notion, Linear, and support inboxes.

Phase 7 — Developer Workflows Planned

cg pipe — composable clipboard pipelines. Git-aware mode that auto-captures commits and branch names. Tmux/zellij per-pane isolation. cg replay to re-execute captured commands.

Phase 8 — Enterprise & Scale Planned

Multi-user mode with role-based access. Audit logging for compliance. Remote clipboard sync via SSH tunnels (zero-trust). Configurable retention per project. Performance hardening for 100K+ item histories.
